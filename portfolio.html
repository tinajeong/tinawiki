🧭 포트폴리오

정산시스템 리빌딩

1. 프로젝트 개요

SQL Server Stored Procedure에 의존하던 27년 된 정산 엔진을 분리·재구성해 Spring Batch 기반 자동화 플랫폼으로 옮겼습니다. 결제 정산 데이터 흐름(수집→검증→집계→지급)을 재정의하고, 운영 가시성·알림·재처리 체계를 통합해 장애 대응 시간을 1분 이내로 줄였습니다.

아키텍처 전환은 3개월 내 월 3천만 건 거래를 병렬 처리하는 목표로 진행됐으며, 금융 보안 및 회계 감사를 충족하도록 데이터 계층과 운영 절차를 함께 설계했습니다.

2. 아키텍처 다이어그램

flowchart LR
  Partners[제휴사 정산 데이터] --> Scheduler[Jenkins & GitHub Actions]
  Scheduler --> Batch[Spring Batch Jobs]
  Batch --> OracleDB[(Oracle 19c 정산 원장)]
  Batch --> Postgres[(PostgreSQL 가시화 스토어)]
  Postgres --> Reconcile[자동 대사 파이프라인]
  Reconcile --> Alerts[Telegram 실시간 알림]
  OracleDB --> Dashboard[Metabase 대시보드]
  Postgres --> Dashboard

3. 주요 기술적 접근

  • Stored Procedure 단위를 도메인 모듈(Core·Domain·Batch)로 해체하고, Step·Flow·Partitioner 조합으로 배치 파이프라인을 재설계해 단계별 검증을 코드화했습니다.
  • Job Parameter, Retry, Skip, Backoff 정책을 명시해 결제 상태별 재처리 시나리오와 롤백 절차를 자동화했습니다.
  • Jenkins 파이프라인과 GitHub Actions 워크플로를 연동해 테스트·정적 분석·배포·스케줄링을 코드 기반으로 관리했습니다.
  • Metabase·Telegram·Slack을 결합해 처리량, 오류율, 정합성 지표를 실시간 대시보드와 알림 메시지로 배포했습니다.

4. 성과 지표

  • 배치 처리 속도 50% 향상 (2.4시간 → 1.2시간)
  • 정산 집계 완료 시간 97% 단축 (180분 → 6분)
  • 장애 감지·대응 시간 1분 이내 확보, 재처리 의존도 0건 유지
  • 월 3천만 건 거래를 무중단 병렬 처리, 월간 운영 인력 투입 시간 40% 절감

6. 역할과 협업

  • 백엔드 총괄로 도메인 분석, 배치 설계, 코드 리뷰, 운영 자동화를 리드했습니다.
  • 재무·운영팀과 주 2회 워킹 세션을 운영해 정산 규칙과 감사 대응 요구사항을 정의했습니다.
  • QA/보안팀과 Checkpoint를 마련해 배치 시나리오 테스트, 암호화 키 로테이션, 장애 대응 리허설을 수행했습니다.
