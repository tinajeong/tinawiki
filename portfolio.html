🧭 포트폴리오

정산시스템 리빌딩

1. 프로젝트 개요

SQL Server Stored Procedure 기반으로 운영되던 27년 된 정산 구조를 Spring Batch 기반 자동화 플랫폼으로 재구축했습니다. 결제 정산 데이터의 집계·검증·지급을 병렬화하고, 운영 지표와 알림 체계를 통합해 안정성을 확보했습니다.

2. 아키텍처 다이어그램

flowchart LR
  Partners[제휴사 정산 데이터] --> Scheduler[Jenkins & GitHub Actions]
  Scheduler --> Batch[Spring Batch Jobs]
  Batch --> OracleDB[(Oracle 19c 정산 원장)]
  Batch --> Postgres[(PostgreSQL 가시화 스토어)]
  Postgres --> Reconcile[자동 대사 파이프라인]
  Reconcile --> Alerts[Telegram 실시간 알림]
  OracleDB --> Dashboard[Metabase 대시보드]
  Postgres --> Dashboard

3. 주요 기술적 접근

  • Stored Procedure 로직을 Spring Batch 멀티모듈 구조로 분리하고 Chunk/Partitioner 전략을 적용했습니다.
  • Job Parameter, Retry, Skip 정책을 정의해 결제 상태별 재처리 안전망을 마련했습니다.
  • Jenkins와 GitHub Actions를 연동해 배포·스케줄링·메트릭 수집을 파이프라인으로 자동화했습니다.
  • Metabase와 Telegram을 결합해 처리량·오류율·정합성 지표를 실시간으로 시각화하고 알림했습니다.

4. 성과 지표

  • 배치 처리 속도 50% 향상
  • 정산 집계 완료 시간 97% 단축
  • 장애 대응 시간 1분 이내 확보
  • 월 3천만 건 거래를 무중단으로 병렬 처리
