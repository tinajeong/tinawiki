홈 소개 경력 포트폴리오 글 연락

요약

대용량 데이터를 다루는 배치 시스템과 마이데이터 플랫폼을 설계·구축·운영해왔습니다. 레거시 배치를 현대화하고 테스트 자동화·데이터 자동 보정처리·운영 가시화처리를 통해 서비스 안정성과 팀 생산성을 함께 끌어올렸습니다. 표준화된 개발·운영 문서와 대시보드를 통해 신규 인력 온보딩과 장애 대응 속도를 체계적으로 개선했습니다.

경력기술서 마스터 블록

거래 집계 시스템 리빌딩 / 백엔드 개발자 @ 다날

27년간 유지된 SQL Server 기반 배치 엔진은 지연 및 단일 장애 지점(SPOF)을 만들어 운영 효율을 떨어뜨렸습니다. 그로인해 잦은 수기처리로 운영팀의 업무 부담이 가중되는 문제가 있었습니다. 이를 해결하기 위해 Stored Procedure를 기능 단위로 해체하고 Spring Batch 5 기반 및 멀티모듈(Core·Domain·Batch) 구조로 재설계했습니다. 이 과정에서 비즈니스 로직을 테스트 가능한 코드로 전환하고, 수집·검증·집계·처리 단계를 Chunk와 Partitioner 전략으로 병렬화하여 대용량 데이터를 안정적으로 처리할 수 있도록 했습니다. 또한 Step Listener 기반 재시도·스킵 정책을 설계하여 예외 상황에서도 일관된 복구가 가능하게 했습니다. Jenkins 파이프라인과 GitHub Actions, SonarQube를 연동하여 배포와 품질 검증을 코드 기반으로 자동화했습니다. 결과적으로 전체 처리 속도는 50% 개선되었으며(ASIS정산거래내역 생성시간과 하루에 도는 정산거래내역 총 실행시간), 실시간 알람 세분화 및 데이터 자동 보정 처리를 통해 운영 효율성을 크게 높였습니다.

기술스택 Java 17, Spring Batch 5, Spring Boot 3, Oracle 19c, PostgreSQL, Jenkins, GitHub Actions, SonarQube, Metabase

핵심 성과

트랜잭션 적재 성능 및 구조 개선

월 3천만 건의 거래 로그를 순차 처리하던 기존 구조를 병렬 처리 아키텍처로 전환했습니다. 데이터 생성부터 검증, 저장까지의 단계를 명확히 분리해 유지보수성과 확장성을 모두 확보했으며, 모듈화된 구조 덕분에 결제 규칙 변경 시 배포 단위를 줄이고 대응 시간을 1일에서 수시간으로 단축했습니다.

데이터 검증 자동화 및 효율화

수동으로 점검하던 거래 검증 과정을 자동화하기 위해 No Offset 및 Clustered Index를 적용하여 조회 성능을 최적화하고 외부 시스템과 연동되는 검증 파이프라인과 알림 체계를 새로 구축했습니다. 이를 통해 170초에서 4초로 단축되었으며, 이상 거래 발생 시 즉시 탐지·대응할 수 있는 운영 환경을 마련했습니다.

정산 처리 안정성 강화

데이터 오류나 중복으로 인한 중단 문제를 해결하기 위해 자동 스킵·중복 방지 정책을 배치 파이프라인에 적용하고, 데이터 정합성 검증 결과를 기반으로 당일 처리 데이터 보정 절차를 자동화했습니다. 그 결과 일일 처리 성공률을 100%로 유지하며, 무중단 정산 환경을 구축했습니다.

월말 마감 처리 효율화

월말 마감 시점에 데이터가 단일 큐에 집중되어 지연이 발생하던 문제를 해결했습니다. Chunk 기반 병렬 처리 파이프라인을 구성하고 상태 전환을 자동 감지하여 결과 반영 시간을 단축했습니다. 이를 통해 마감 집계 속도가 30% 향상되었고, 마감 후 1분 내 운영 현황이 자동 갱신되어 실시간 일관성을 유지할 수 있게 되었습니다.

파트너 인터페이스 구조 고도화

기존에는 수작업으로 파일을 전송(SCP)하던 프로세스를 API 기반 자동화로 전환했습니다. RESTful 업로드 API와 배치 파이프라인을 설계하고, Caffeine Cache를 활용하여 대규모 동적 데이터를 효율적으로 관리했습니다. 또한 파일 비교·검증 Job을 추가해 전송 누락을 방지했으며, 이를 통해 운영 대응 시간을 60% 절감했습니다.

테스트 환경 및 모듈 구조 개선

단일 모듈·공유 DB로 인한 테스트 불안정성을 해소하기 위해 Modular Monolith 아키텍처를 도입했습니다. H2와 PostgreSQL을 조합한 테스트 환경을 마련하고 EntityFixtureGenerator·Job 파라미터 Proxy Bean 패턴을 통해 테스트 코드의 재사용성을 높였습니다. 빌드 시간은 25% 단축되었고, 신규 인력의 온보딩 기간을 1주에서 3일로 줄였습니다.

운영 모니터링 체계 강화

배치 실행 단계별 가시성과 표준화된 예외 코드가 부족했던 문제를 해결했습니다. Telegram 알림 시스템을 구축하고 Custom Exception 기반 코드 체계를 표준화했으며, Metabase를 통해 70개 이상의 데이터 검증 대시보드를 구성했습니다. 그 결과 장애 인지와 조치 시간이 평균 3분 이내로 단축되었습니다.

자동 검증·리포트 파이프라인 / 백엔드 개발자 @ 다날

기존에는 검증 보고서와 정합성 점검이 수동으로 이루어져, 이상 탐지와 대응이 지연되는 문제가 있었습니다. Spring Batch Step Listener와 룰 엔진을 결합하여 거래 상태별 검증 리포트를 자동으로 생성하고 PDF 및 Slack 템플릿으로 배포하는 시스템을 구축했습니다. 또한 Metabase를 통해 처리량, 오류율, 재시도, 정합성 지표를 실시간으로 시각화하고 SLA 위반 시 조건부 서식을 적용하여 가시성을 높였습니다. Telegram Bot API를 활용해 Job과 Step 이벤트, 임계값 초과, 재시도 결과를 실시간으로 알림으로 전송했고, 재처리·승인·배포 과정을 표준 운영 문서로 정리해 담당자 변경 시에도 1시간 이내에 온보딩이 가능하도록 체계를 마련했습니다. 이로써 반복 검증 시간이 3시간에서 12분으로 단축되고, 이상 탐지부터 대응까지 평균 5분 이내로 수행할 수 있게 되었습니다.

기술스택 Spring Batch, Java 17, Metabase, Telegram Bot API, Jenkins, GitHub Actions, Slack API

마이데이터 백엔드 플랫폼 / 백엔드 개발자 @ 쿠콘

마이데이터제공자의 입장에서 정보제공자의 업권별 데이터 포맷과 API 규격이 다르고, 각 회사에서 이해하는 데이터의 의미가 상이하여 품질 검증 과정이 병목 구간으로 작용했습니다. 이를 해결하기 위해 은행·통신·보험·캐피탈 4개 업권의 수집 모듈을 전략 패턴으로 재설계하여 공통 DTO, 검증/ 예외 처리를 제공했습니다. 거래 및 계좌 무결성 검증 로직을 배치 파이프라인으로 구현하고, 오류 유형을 자동 태깅하는 품질 대시보드를 구축했습니다. 개인정보 보호 강화를 위해 AES-256/SHA-256 암호화와 전용 키 로테이션 스케줄러를 도입하여 보안성을 강화했고, 배포/장애 대응/고객 VOC를 위한 Runbook을 정리해 DevOps 대응 속도를 표준화했습니다. 결과적으로 신규 기관 연동 및 안정화 기간이 평균1.5주에서 2시간 내외로 단축되었고, QA 오류 대응 시간을 절반이상 줄일수 있었습니다.

기술스택 Spring Boot 2.6, Java 8, PostgreSQL, AES-256/SHA-256, JUnit 5, GitLab CI, 내부 대시보드
