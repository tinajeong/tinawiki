정산시스템 리빌딩 / 백엔드 개발자 @ 다날

문제 정의 27년간 유지된 SQL Server Stored Procedure 구조가 처리 한계와 운영 리스크를 초래했습니다.

  • Spring Batch 5 기반 멀티모듈 아키텍처(Core·Domain·Batch)를 설계해 로직을 데이터 중심 구조로 분리했습니다.
  • Partitioner+Chunk 병렬화와 재시도 정책을 적용해 검증·집계·지급 단계를 독립 배치로 리파인했습니다.
  • Oracle 19c와 PostgreSQL을 병행 사용해 정산 원장과 가시화 레이어를 분리했습니다.
  • Jenkins·GitHub Actions·SonarQube를 연동해 코드 품질 검증과 스케줄링을 자동화했습니다.

결과 처리 속도 50% 향상, 집계 완료 시간 97% 단축, 장애 대응 시간을 1분 이내로 축소했습니다.

기술스택 Java 17, Spring Batch 5, Spring Boot 3, Oracle 19c, PostgreSQL, Jenkins, GitHub Actions, Metabase

자동화 대사 파이프라인 / 백엔드 개발자 @ 다날

문제 정의 수기 기반 정산 검증과 늦은 알림으로 데이터 이상을 사후에만 발견했습니다.

  • Spring Batch Listener와 비즈니스 룰을 결합해 집계 결과를 템플릿 기반 대사 리포트로 자동 생성했습니다.
  • Metabase 대시보드에 주요 지표(처리량, 오류율, 재시도, 정합성)를 집계해 추세를 시각화했습니다.
  • Telegram Bot API로 Job/Step 단위 이벤트를 전송해 운영팀이 즉시 대응하도록 했습니다.
  • 가이드 문서를 작성해 재처리·승인·배포 절차를 표준화했습니다.

결과 반복 검증 시간을 시간 단위에서 분 단위로 단축했고, 이상 징후를 실시간으로 공유 가능한 체계를 마련했습니다.

기술스택 Spring Batch, Java 17, Metabase, Telegram Bot API, Jenkins, GitHub Actions

마이데이터 백엔드 플랫폼 / 백엔드 개발자 @ 쿠콘

문제 정의 업권마다 다른 데이터 포맷과 보안 요구로 인해 신규 제휴사 온보딩과 품질 관리를 신속히 처리하기 어려웠습니다.

  • 은행·통신·보험·캐피탈 수집 로직을 전략 패턴 모듈로 분리해 공통 DTO와 검증 규칙을 통일했습니다.
  • 거래·계좌 무결성 검증 로직과 대시보드를 구축해 오류 케이스를 자동 분류하고 가시화했습니다.
  • AES-256/SHA-256 암호화와 배치 전송 파이프라인을 설계해 개인정보 전달을 자동화했습니다.
  • 배포·장애 대응 프로세스를 문서화하고 정기 리허설을 통해 운영 기준선을 세웠습니다.

결과 신규 기관 연동과 QA 오류 대응 속도를 단축했고, 개인정보 보호 감사 대응 시간을 안정화했습니다.

기술스택 Spring Boot 2.6, Java 8, PostgreSQL, AES-256/SHA-256, JUnit5, Internal Dashboard
