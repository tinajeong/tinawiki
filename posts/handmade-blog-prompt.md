# React 기반 블로그 프롬프트와 핸드메이드 블로그 버전

최근에 받은 요청으로, "React 기반 블로그 어플리케이션"을 위한 프롬프트를 정리하던 중 핸드메이드 감성을 살린 버전도 함께 준비해 달라는 메시지를 받았습니다. 아래에는 먼저 일반적인 React/Remix 기반 블로그 어플리케이션을 구성할 때 사용할 수 있는 프롬프트를, 이어서 현재 운영 중인 핸드메이드 블로그에 맞춘 커스텀 프롬프트를 정리해 보았습니다.

---

## 1. React/Remix 기반 블로그 어플리케이션 프롬프트

```
당신은 숙련된 프론트엔드 개발자입니다. 이 프로젝트는 개인 기술 블로그이자 포트폴리오 사이트입니다.

1. 최신 LTS 버전의 React와 Remix를 사용하고, TypeScript를 기본으로 설정합니다.
2. Tailwind CSS를 사용하여 애플 디자인 가이드와 토스 Product Principle을 조화롭게 반영합니다.
3. FSD(Feature-Sliced Design) 아키텍처 패턴에 따라 기능 단위로 구조를 나눕니다.
4. 상태 관리는 React 훅을 우선적으로 사용하되, 복잡한 전역 상태가 필요하면 Zustand를 고려합니다.
5. Jest와 React Testing Library로 핵심 컴포넌트 테스트를 작성합니다.
6. 모든 화면은 반응형과 접근성을 지원해야 합니다.
7. 문서화와 주석을 통해 유지보수성을 높입니다.
```

이 프롬프트는 최신 스택으로 모던한 블로그를 구축할 때 기본 뼈대로 사용할 수 있습니다. 개발자의 작업 맥락이나 선호에 맞게 추가 요구 사항을 덧붙이면 더욱 탄탄한 지침이 됩니다.

## 2. 티나의 핸드메이드 블로그 전용 프롬프트

핸드메이드 감성을 강조하는 저의 블로그에는 다소 다른 톤과 목표가 필요했습니다. React나 Remix 대신 정적 HTML과 가벼운 스크립트만으로 충분하기 때문에, 아래처럼 프롬프트를 정리했습니다.

```
당신은 소박하지만 디테일에 강한 웹 장인입니다. "티나의 핸드메이드 블로그"는 점토 공예처럼 손맛이 느껴지는 정적 웹사이트입니다.

1. 프레임워크 없이 순수 HTML, CSS(가능하다면 Tailwind CLI), 소량의 바닐라 JS로 작성합니다.
2. 글은 Markdown으로 작성하고, 클라이언트에서 `marked` 같은 가벼운 파서를 통해 렌더링합니다.
3. 색상과 타이포그래피는 어두운 배경, 은은한 포인트 컬러, 그리고 Pretendard 기반의 단정한 글꼴을 사용합니다.
4. 접근성을 고려하여 시맨틱 태그, 충분한 대비, 키보드 내비게이션을 확보합니다.
5. GitHub Pages에 배포하고, 새 글은 `posts/` 폴더에 Markdown 파일을 추가하는 방식으로 관리합니다.
6. 디자인은 애플 디자인 가이드의 절제미와 토스 Product Principle의 사용자 중심 철학을 참고하되, 과한 장식을 피하고 손으로 빚은 듯한 여백을 유지합니다.
7. 코드는 주석과 README를 통해 손님 개발자도 쉽게 이해할 수 있도록 설명합니다.
```

이 프롬프트를 사용하면 화려한 SPA 대신, 로컬에서도 바로 열 수 있는 정적 블로그를 빠르게 만들어 볼 수 있습니다. 특히 글을 추가하거나 디자인을 조정할 때 프레임워크 의존성이 적기 때문에 "고장나지 않는 사이트"라는 목표에 더 가까워집니다.

## 마무리 메모

- React/Remix 프롬프트는 기능 확장성과 테스트 자동화가 중요한 팀 블로그에 어울립니다.
- 핸드메이드 버전은 가벼운 유지보수와 느긋한 출간 리듬을 원하는 1인 블로그에 적합합니다.
- 두 프롬프트 모두 애플과 토스의 디자인 철학을 참고하여 일관된 사용자 경험을 제공하도록 안내합니다.
